# 徐振伟智能体 (Xu Zhenwei Agent) — 系统设计书

**版本**: 2.0 | **日期**: 2026-06-17 | **作者**: 徐振伟 + Claude Code

---

## 一、系统概述

徐振伟智能体是一个基于 56 技法和 18 配方的 AI 分析型对话系统。用户输入问题 → AI 推荐最合适的分析技法 → 用户确认后执行 → 输出带技法标签的结构化结果。

**技术栈**:
- 前端: 纯 HTML/CSS/JS（SPA，无框架依赖）
- 后端: Spring Boot + Claude API
- 部署: 本地服务器 `http://localhost:8080`

---

## 二、核心功能

### 2.1 分析模式

| 模式 | 说明 |
|------|------|
| 💬 自由对话 | 不限定技法，AI 自由回答 |
| 🤖 智能推荐 | 默认模式。输入问题 → AI 推荐 5 个技法（带匹配度%）→ 用户选择执行 |
| 🔧 56技法 | 按4大分类×12子类折叠展示，点击直接执行 |
| 📦 18配方 | 预设技法组合：创业验证/内容创作/客户获取/战略规划/提案打磨/农业品牌/深度诊断 |

### 2.2 56技法分类

```
快速产生灵感（17个）
  ├── 用AI特有力量思考 (001-006)
  ├── 用自由发想思考 (007-012)
  └── 用逻辑发想思考 (013-017)

打磨完善创意（15个）
  ├── 让思考发展 (018-023)
  ├── 让思考具体化 (024-026)
  └── 让思考验证 (027-032)

落地执行创意（6个）
  ├── 思考传播方式 (033-036)
  └── 思考执行策略 (037-038)

找到思考切入点（18个）
  ├── 分析课题得启示 (039-043)
  ├── 分析烦恼得启示 (044-046)
  ├── 分析人得启示 (047-053)
  └── 预测未来得启示 (054-056)
```

### 2.3 文件上传

- **支持格式**: .md/.txt/.csv/.json/.xml/.yml/.yaml/.html/.log/.java/.py/.js/.ts/.css/.pdf
- **数量限制**: 最多 10 个文件
- **大小限制**: 单文件 ≤ 5MB
- **交互流程**:
  1. 用户点击「📁 选择文件」或「📂 选择文件夹」
  2. 选中文件后显示为标签（显示文件名 + 大小 + ✕ 删除按钮）
  3. 文件**不立即发送**，等待用户输入文字
  4. 用户输入文字 + 按 Enter → 文件内容 + 文字 + 技法 统一发送
- **文件夹选择**: 使用 `webkitdirectory` 属性，兼容 Chrome/Edge

### 2.4 文件选择按钮实现

采用 label 包裹 input[type=file] 的标准模式：
```html
<label class="fbtn file">
  <span>📁 选择文件</span>
  <input type="file" multiple>
</label>
```
```css
.fbtn { position: relative; overflow: hidden; }
.fbtn input[type=file] {
  position: absolute; top: 0; left: 0;
  width: 100%; height: 100%;
  opacity: 0; cursor: pointer;
}
```

### 2.5 技法标签（2026-06-17 新增）

每次分析结果的气泡顶部，自动显示使用的技法标签：

- 🔧 **技法标签**（紫色）：使用单个 56 技法时显示，如 `🔧 技法: 用AI特有力量思考`
- 📦 **配方标签**（绿色）：使用 18 配方时显示，如 `📦 配方: 📦 创业验证`

实现方式：
- 全局变量 `currentTechLabel` 追踪当前执行的技法
- `finish()` 函数在结果气泡中插入 `tech-badge` 元素

## 三、交互流程

### 3.1 透明推荐流程（默认模式）

```
用户输入文字（可选: +上传文件）
        │
        ▼
┌─────────────────────┐
│ 读取文件内容（如有） │
│ 构建完整 prompt      │
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│ POST /api/recommend  │──→ 返回 Top5 技法卡片
│ {message: prompt}    │    (id + name + reason + confidence%)
└─────────────────────┘
        │
        ▼
┌─────────────────────────────────────┐
│ 显示推荐卡片:                        │
│  🔑 关键词  [tag1] [tag2]           │
│  📊 001 技法名  原因 95%            │
│  📊 003 技法名  原因 88%            │
│  ...                                │
│  [▶ 用推荐执行] [📦 用组合配方]     │
│  [🔄 换一批推荐]                    │
└─────────────────────────────────────┘
        │ 用户点击
        ▼
┌─────────────────────┐
│ POST /api/agent/think│──→ SSE 流式输出
│ {message,techniqueId}│    推理过程 + 最终结果
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│ 结果气泡:            │
│ ┌─────────────────┐ │
│ │ 🔧 技法: XXX    │ │ ← 2026-06-17 新增
│ │ (分析内容...)   │ │
│ │            [📥] │ │
│ └─────────────────┘ │
└─────────────────────┘
```

### 3.2 直接执行流程

用户从侧边栏点击 56 技法或 18 配方 → 输入问题 → 直接执行，跳过推荐

## 四、API 接口

| 端点 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 返回 index.html |
| `/api/techniques` | GET | 获取 56 技法列表（含分类） |
| `/api/recommend/quick` | POST | 快速返回 Top5 技法推荐 |
| `/api/agent/think` | POST | 执行分析，SSE 流式输出 |
| `/api/export` | POST | 导出 PPT/Word/Excel |
