## 一、系统概述

徐振伟智能体是一个基于 **106种技法**（56创意技法 + 50 AI独学TIPS）的 AI 分析型对话系统。用户输入问题 → AI 推荐最合适的分析技法 → 用户确认后执行 → 输出带技法标签的结构化结果。

**技术栈**: Spring Boot 3.4.7 + Spring AI 1.0.3 + DeepSeek API + H2 / 纯 HTML/CSS/JS SPA 前端

**核心设计理念**：两个独立维度自由组合

```
        思考深度 (Layer 1)          分析方式 (Layer 2)
    🧠 深度思考 ←→ ⚡ 快速      💬 自由对话 ←→ 🔧 技法分析
    
    4 种组合模式:
    ① 🧠+💬 深度自由   ② 🧠+🔧 深度技法
    ③ ⚡+💬 快速自由   ④ ⚡+🔧 快速技法
```

## 二、深度思考模式 (Layer 1)

### 2.1 模式定义

| 属性 | 🧠 深度思考 (默认) | ⚡ 快速模式 |
|------|-------------------|------------|
| **后端** | DeepSeek Reasoner（推理模型）| DeepSeek Chat（快模型）|
| **流程** | 2阶段：推理 → 精炼整理 | 1阶段：直接回答 |
| **耗时** | 10-60秒 | 2-8秒 |
| **Token消耗** | 高（~8K tokens）| 低（~2K tokens）|
| **适用场景** | 复杂分析、方案策划、战略规划 | 简单问答、事实查询 |
| **可见推理** | ✅ 显示推理过程 | ❌ 无推理展示 |
| **技法推荐** | ✅ 显示推荐卡片 | ⬜ 跳过推荐 |

### 2.2 按钮交互

```
⚡ 快速按钮:
├── 关闭(默认): 灰色边框，"⚡ 快速"
├── 开启: 金色渐变背景+发光，"⚡ 快速开"
└── 状态标签联动: "🧠 深度思考" ↔ "⚡ 快速模式"
```

### 2.3 后端分流逻辑

```java
// AgentController.think() 分流
if (fastMode && !hasTechnique) → agentEngine.quickChat()      // ⚡+💬
if (fastMode && hasTechnique)  → techniqueExecutor.execute()  // ⚡+🔧 
if (!fastMode && hasTechnique) → deepThink + techniqueExec    // 🧠+🔧
if (!fastMode && !hasTechnique) → deepThink                   // 🧠+💬 (默认)
```

| 盲区 | 描述 | 农业场景中的后果 |
|------|------|----------------|
| **群体共创引导** | 所有技法均为"单用户+AI"，无多人协作模式 | 无法在15-30人培训班中集体使用一个技法 |
| **物理空间联动** | 无实景互动设计 | 不能"站在大棚里用AI分析当前作物" |
| **长期行为改变** | 有机制化但偏AI使用机制，非行为养成 | "学完回去不执行"无法被系统解决 |
| **跨语言适配** | 纯中文场景 | 无法直接检索日语六次产业化原始文献 |
| **数据采集闭环** | 输出是终点，不是下轮起点 | 学员反馈无法自动汇聚成课程改进报告 |

### 8.2 建议新增的5条技法

#### 新增A：隐喻转化法（改编自 TIPS-44）
- **位置**：第一部"快速产生灵感"
- **理念**：复杂概念不是"解释清楚"的，是"找到对的比喻"后自然被理解的
- **步骤**：列出核心概念→从3个领域各造比喻→测试映射准确度→选出最佳金句

#### 新增B：认知偏差体检法（改编自 TIPS-42）
- **位置**：第二部"打磨完善创意"
- **理念**：创意最致命的漏洞不在方案本身，而在创造者的思维习惯
- **步骤**：输入过去决策→AI识别偏差模式→给出纠正练习→标注受偏差影响的方案部分

#### 新增C：能力取舍策略法（改编自 TIPS-25）
- **位置**：第三部"落地执行创意"
- **理念**："不做什么"比"做什么"更重要
- **步骤**：列出全部能力/资源→投入产出评估→三类分拣→最短路径制定

#### 新增D：游戏化学兵法（改编自 TIPS-32）
- **位置**：第三部"落地执行创意"
- **理念**：枯燥的执行过程通过游戏机制获得持续动力
- **步骤**：目标→RPG任务→XP系统→成就里程碑→Boss战→仪式感奖励

#### 新增E：本质提问法（改编自 TIPS-50）
- **位置**：第四部"找到思考切入点"（作为元方法）
- **理念**：人类核心能力不是"回答"而是"提出本质性问题"
- **步骤**：禁止问"怎么做"→5个本质提问框架→转化AI指令→注入个人灵魂→跳出常规产出
