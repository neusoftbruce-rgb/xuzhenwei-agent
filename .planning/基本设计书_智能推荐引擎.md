## 2. 推薦引擎：核心アルゴリズム

### 2.1 三层判定流程

```
Step 1: 复杂度分类 (0ms, 本地关键词)
        用户输入 → classify() → ComplexityLevel
Step 2: 置信度截断 (0ms, 分数分布分析)
        排序结果 + ComplexityLevel → determineCutoff() → 截断数
Step 3: 结果返回
        截断后的技法列表 + complexity元数据 → API响应
```

### 2.2 复杂度等级定义

| 等级 | 判定条件 | 推荐上限 | 典型问题 |
|------|---------|---------|---------|
| SIMPLE_FACTUAL | 定义/事实询问型关键词 | 1-2条 | "什么是酵素菌？" |
| SINGLE_DOMAIN | 默认等级 | 2-4条 | "大棚病害怎么治？" |
| COMPLEX_MULTI | 50字以上 或 多领域关键词 | 3-5条 | "高糖番茄怎么种+怎么卖？" |
| COMPREHENSIVE | 全面诊断型 | 5-8条 | "帮我全面分析农场商业模式" |

### 2.3 置信度断崖算法

```
输入: sortedResults (按分数降序), ComplexityLevel
输出: cutoffIndex

算法:
  cutoff = level.minRecommend
  FOR i = minRecommend-1 TO sorted.size-1:
    gap = sorted[i].score - sorted[i+1].score
    IF gap > 0.15 (断崖阈值):
      cutoff = i+1; BREAK
    cutoff = min(cutoff+1, level.maxRecommend)
  RETURN min(cutoff, level.maxRecommend)
```

断崖阈值 0.15 的根拠：当前分数范围 [0.3, 0.9]，0.15相当于20%的分数跨度。

## 3. 技法溯源：データフロー

### 3.1 前→後 技法信息传递

```
用户发送问题
  → 前端 currentTechLabel = "🔧 技法: [038] 行动计划生成法"
  → SSE流结束，finish() 在气泡插入 tech-badge div
  → DOM中保存完整技法信息

用户点击"导出"
  → 前端从 msg.ai 气泡的 .tech-badge 提取技法文本
  → POST /api/export { content: "...", format: "WORD", 
                        techniqueLabel: "[038] 行动计划生成法 · 深度思考" }
  → 后端在文档末尾追加技法溯源章节
```

### 3.2 后端导出接口变更

**现有接口** (ExportController):
```java
record { content, format, title }
```

**新接口**:
```java
record { content, format, title, techniqueLabel }  // techniqueLabel 可选
```

### 3.3 技法溯源章节テンプレート

```
--- (分隔线)
📋 分析方法溯源
本次分析使用了徐振伟智能体的以下技法：
• [038] 行动计划生成法 — 把想法变成待办清单
• [001] 跨界特征联想法 — 从动物身上找灵感
分析模式: 🧠 深度思考
生成时间: 2026-06-17 15:30
Powered by 徐振伟智能体 v2.0
```
