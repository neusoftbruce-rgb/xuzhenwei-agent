## 4. API 设计（新增）

复用现有 API，新增认证接口：

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/wechat-login` | 微信登录（code → JWT） |
| GET | `/api/auth/me` | 获取当前用户信息 |
| POST | `/api/agent/think` | **已有，复用** |
| GET | `/api/sessions` | **已有，复用** |
| GET | `/api/techniques` | **已有，复用** |
| POST | `/api/export` | **已有，复用** |

### 新增：微信登录 API

```
POST /api/auth/wechat-login
Request:  { "code": "081x4z0w3..." }
Response: { "token": "eyJhbG...", "user": { "openid": "oABC...", "nickname": "徐振伟" } }
```

### SSE 流式适配

微信小程序 `wx.request` 不支持标准 EventSource，需要 `enableChunked: true` 手动解析：

```javascript
// 小程序 SSE 适配
wx.request({
  url: 'https://api.xxx.com/api/agent/think',
  method: 'POST',
  enableChunked: true,  // 关键
  data: { message, sessionId, ... },
  success: (res) => {
    // res.data 是完整响应，但在 onChunkReceived 中逐步接收
  }
})

// 需要在 wx.request 的 success 回调中监听
// 或使用 wx.request 的 onChunkReceived 回调（基础库 2.20.1+）
```

## 5. 数据模型

### 5.1 复用现有（不改）

```
sessions (会话)
message_records (消息)
session_outputs (产出)
```

### 5.2 新增（小程序端）

```javascript
// 小程序本地缓存
{
  token: "eyJhbG...",           // JWT
  currentSessionId: "uuid",     // 当前会话
  draftText: "用户正在输入...",   // 草稿
}
```

### 5.3 新增（后端）

```sql
-- 用户表（微信登录）
CREATE TABLE users (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  openid VARCHAR(64) UNIQUE NOT NULL,
  unionid VARCHAR(64),
  nickname VARCHAR(100),
  avatar_url VARCHAR(500),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 为 sessions 表增加 user_id 外键
ALTER TABLE sessions ADD COLUMN user_id BIGINT REFERENCES users(id);
```

## 8. 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 框架 | 原生开发 | 包体积最小，不用第三方框架 |
| UI | 自写 CSS | 设计一致性，不依赖 WeUI |
| 语音 | wx.getRecorderManager | 微信原生，识别率最高 |
| SSE | enableChunked | 微信基础库 2.20.1+ 原生支持 |
| 登录 | wx.login + JWT | 最简单，无需手机号 |
| Markdown | 自写简易渲染 | 避免引入大型库，只支持 h2/h3/bold/list |
