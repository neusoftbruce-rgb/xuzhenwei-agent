# 徐振伟智能体 微信小程序 — 要件定义书

> 版本: v1.0 | 日期: 2026-06-18 | 作者: 徐振伟

---

## 1. 业务要件

### 1.1 背景

徐振伟智能体（Web版）已上线运行，提供56种创意技法 + AI深度对话能力。但农业培训现场、田间地头、学员拜访等场景下，打开电脑不方便。需要小程序版本，**手机随时聊、随时问、随时查**。

### 1.2 目标用户

| 用户群 | 使用场景 | 频率 |
|--------|----------|------|
| 农业培训学员 | 课间提问、课后复习技法 | 每周2-3次 |
| 农场主/棚主 | 田头遇到病害/管理问题即时咨询 | 按需 |
| 徐振伟本人 | 演示技法、现场调研录音转分析 | 每日 |

### 1.3 业务目标

- **30秒内完成一次咨询**：打开小程序→说话/打字→得到AI分析
- **历史对话随身带**：手机上的对话和电脑端同步（通过后端数据库）
- **语音为主**：田头打字不方便，语音输入是第一交互方式

| ID | 功能 | 优先级 | 说明 |

### 2.3 不做（范围外）

- ❌ 不做的：56技法全部在小程序内展示（太重，只做常用12个+配方）
- ❌ 不做的：离线模式（AI必须在线）
- ❌ 不做的：多语言（只做中文）
- ❌ 不做的：社交分享（v1不做）

## 5. 数据流概要

```
小程序 ──wx.request──▶ Spring Boot API Server
  │                        │
  │  wx.login() → code     │  → 微信服务器换取 openid
  │  ← JWT token           │
  │                        │
  │  POST /api/agent/think │  → DeepSeek API (SSE)
  │  ← SSE stream          │
  │                        │
  │  GET /api/sessions     │  → H2/PostgreSQL
  │  ← JSON                │
```

## 7. 与 Web 版的差异

| 项目 | Web版 | 小程序版 |
|------|-------|----------|
| 交互方式 | 鼠标点击为主 | **触摸 + 语音为主** |
| 侧边栏 | 260px固定 | 滑出抽屉 |
| 技法展示 | 56条全展开 | **常用12条 + 配方** |
| 语音 | 浏览器Web Speech | **微信原生录音API** |
| 文件上传 | 拖拽/选择 | 拍照/相册/聊天文件 |
| 登录 | 无登录 | **微信一键登录** |
| 数据存储 | 本地H2 | 共享PostgreSQL |
