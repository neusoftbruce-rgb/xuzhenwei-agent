# 徐振伟智能体 v2.3 要件定义书

## 1. 背景与目的

### 1.1 现状
徐振伟智能体 v2.1 已完成核心功能：
- 56种创意技法 + 50种AI独学TIPS = 106技法
- 智能推荐引擎（动态截断 + 复杂度分类）
- 技法溯源导出
- 会话持久化
- 文件上传分析
- 语音输入
- 13种配方套餐

### 1.2 问题
通过系统性代码审查，发现以下问题：

**后端Bug（13个）：**
1. recipe执行不尊重deepThink设置
2. ConversationManager缓存淘汰逻辑错误（不维护插入顺序）
3. conversationId解析在复合键场景下脆弱
4. MemosSyncService手动JSON拼接有注入风险
5. TokenSaver缓存用hashCode做key有碰撞风险
6. DomainAdvisor关键词匹配仅限id+name
7. SessionService.saveOutput计数跨会话污染
8. AgentController.deepTechniqueThink丢失对话上下文
9. JWT filter浪费在所有公开端点上
10. ResponseRefiner正则遗漏扩展汉字
11. 异常处理缺少用户友好错误分类
12. 深度思考1.5K截断硬编码不合理
13. 缺少请求频率限制

**前端Bug（11个）：**
1. loadSessions中变量重复声明导致bar引用混淆
2. 首条消息创建会话失败时继续执行无保护
3. 配方执行每次创建新conversationId丢失上下文
4. btoa/unescape已废弃API
5. marked.parse XSS 风险（无HTML过滤）
6. SSE多行data解析不支持
7. 流中断无重试机制
8. 空响应也添加导出按钮
9. openSession用innerHTML渲染历史消息有XSS
10. sessionBar无会话时隐藏后无法自动恢复
11. 多次DOMContentLoaded绑定导致initSpeech/loadSessions重复执行

### 1.3 目标
修复所有已知bug，增加2026年AI智能体最佳实践功能，让用户操作更舒适。

## 3. 非功能要件

### 3.1 性能
- 推荐引擎响应 < 5ms（纯本地计算）
- SSE流式响应首字节 < 500ms
- 前端首屏加载 < 2s

### 3.2 安全
- 所有用户输入经过XSS过滤
- API限流：每分钟最多20次请求
- 文件上传大小限制5MB/个，最多10个
- JWT token过期自动刷新

### 3.3 可用性
- 离线提示：后端不可用时前端显示友好提示
- 错误分级：用户错误 vs 系统错误 vs AI错误
- 操作可撤销：3秒内可撤销发送

## 4. 制约条件

- Java 17 + Spring Boot 3.4.7 + Spring AI 1.0.3
- DeepSeek API（OpenAI兼容）
- H2文件数据库
- 单机部署，支持后续迁移PostgreSQL
- 前端纯HTML/CSS/JS，不引入构建工具
- 兼容Chrome/Firefox/Safari/Edge最新版
