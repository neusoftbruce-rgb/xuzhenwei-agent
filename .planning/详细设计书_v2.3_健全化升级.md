## 2. 新增功能设计

### 2.1 UX-03: 取消生成按钮

**前端实现：**
- 发送按钮右侧新增 ⏹ 停止按钮，默认隐藏
- 发送时 `showStopButton()` → 隐藏"发送"显示"⏹"
- 点击调用 `stopGeneration()` → `abortController.abort()`
- streamToMsg中的fetch携带`signal: abortController.signal`
- AbortError特殊处理：显示"⏹ 已取消"
- 结束后 `hideStopButton()` → 恢复"发送"按钮

**后端：**（无需修改，前端断开SSE连接即可）

### 2.2 UX-01: 输入历史导航

**实现：**
- 闭包模块 `inputHistory`：维护 `history[]` 和 `pos` 指针
- send()时调用 `inputHistory.push(text)`
- handleKey中：↑键 `getPrev()`, ↓键 `getNext()`
- 最多保存100条

### 2.3 安全增强

**safeMarked 渲染器：**
- 覆盖marked.Renderer.html()：所有原始HTML经过escapeHtml()转义
- 覆盖marked.Renderer.link()：只允许http/https链接，其他替换为#blocked
- 兜底异常捕获：marked.parse失败时fallback到escapeHtml

## 3. 数据流变更

### 3.1 请求取消流程
```
用户点击 ⏹ → stopGeneration()
  → abortController.abort()
  → fetch抛出 AbortError
  → streamToMsg显示 "⏹ 已取消"
  → finish()不执行（流中断）
  → hideStopButton()恢复界面
```

### 3.2 会话创建降级流程
```
send() → POST /api/sessions
  ├─ 成功 → currentSessionId = session.id
  └─ 失败 → currentSessionId = "local_<timestamp>_<random>"
           → 显示 "⚠️ 会话服务暂不可用，使用本地模式"
           → 消息照常发送（API不强制要求sessionId）
```

## 4. API变更

### 4.1 需要Content-Type header的fetch
所有 POST 请求的 fetch 调用已携带 `headers: {'Content-Type': 'application/json'}`，新增 `signal: abortController.signal`。
